# 一、项目概述

## 1.1 项目背景

随着我国人口老龄化进程加快，养老机构在床位调度、入住评估、合同与费用管理、请假与退住办理等方面业务量持续增长。传统以纸质台账或分散表格为主的管理方式，容易出现信息孤岛、流程进度不透明、审批衔接困难等问题，难以支撑多角色协同与过程留痕。

在此背景下，本课题面向养老院日常运营管理场景，设计并实现「中州养老管理系统」（工程仓库：`zzyl-project`）。系统以 Web 管理端为核心，将预约来访、入住办理、在住服务、退住清算及协同审批等业务纳入统一平台，提升机构内部的信息化水平与业务闭环能力。

## 1.2 项目意义与建设目标

本项目的建设意义主要体现在三个方面：一是以标准化流程替代口头交接与零散记录，保证入住、退住等关键业务可追踪、可回溯；二是通过工作台统计与协同待办，缩短多岗位协作路径，提高办理效率；三是在业务系统之上引入大模型对话能力，为后续智能问答与辅助决策预留技术基础。

结合课程实践与工程实现，本系统建设目标可概括如下：

1. **业务数字化**：覆盖来访登记、入退住办理、合同与床位、客户档案、请假管理等核心业务。
2. **流程规范化**：入住按五步推进，退住按七步推进，关键节点支持审批通过、拒绝与退回。
3. **协同可视化**：在「我的待办」「我的申请」中汇聚入住、退住、请假等流程单据，便于统一处理与状态查询。
4. **技术可落地**：采用主流前后端分离架构，完成可运行的 B/S 管理端，并集成对象存储与 AI 对话接口。

## 1.3 系统简介

中州养老管理系统采用 **B/S 架构的 Web 管理端**实现，面向养老机构内部管理人员与业务办理人员使用，**非微信小程序、非移动端独立 App**。用户通过浏览器访问前端页面，经登录后进入主框架，完成日常业务操作。

系统前后端分离部署：前端工程为 `zzylvue`（Vue 3），后端工程为 `zzyl`（Spring Boot），默认通过 HTTP 接口交互。登录后可进入工作台查看预约、来访、客户、请假、退住等汇总数据；各业务模块按菜单组织，支持分页查询、详情办理、文件上传及流程步骤推进。系统同时提供基于 Spring AI + 阿里云 DashScope 的聊天接口，以及阿里云 OSS 相关能力，用于支撑智能交互与文件存储扩展。

## 1.4 已实现功能模块概览

依据前端菜单（`menu.js`）、路由配置及后端 Controller，系统当前已实现的主要功能模块如下。

| 模块 | 主要功能 |
|------|----------|
| 登录与个人中心 | 用户登录/退出；个人信息维护 |
| 工作台 | 预约、来访、客户、请假中、退住等业务量统计展示 |
| 来访管理 | 预约登记、来访登记 |
| 入退管理 | 入住申请/入住办理；退住申请/退住办理；详情页按步骤推进 |
| 在住管理 | 合同跟踪与合同详情；床位房型、智能床位视图、房型设置；请假管理与请假详情 |
| 客户管理 | 客户信息查询与维护 |
| 协同工作 | 我的待办、我的申请（汇聚入住/退住/请假流程） |
| 护理相关（路由与后端已实现） | 护理项目、护理计划、护理等级的增删改查及计划项关联 |
| AI 能力 | `/chat01` 一次性对话、`/chat02` 流式对话（DashScope） |

说明：护理相关页面已在前端路由与后端接口中实现，当前主菜单以来访、入退、在住、客户、协同等业务为主；智能床位页面基于楼层—房间—床位数据展示床位占用与状态视图，用于在住床位可视化管理。

## 1.5 技术架构简述

本系统采用前后端分离的分层架构，技术选型与仓库实现一致，主要技术栈如下：

| 层次 | 技术 |
|------|------|
| 前端 | Vue 3、Vue Router、Element Plus、Axios |
| 后端 | Spring Boot 3.5.x（Java 17）、Spring Web |
| 持久层 | MyBatis-Plus、MySQL |
| 文件与存储 | 本地上传接口；阿里云 OSS SDK |
| 人工智能 | Spring AI、Spring AI Alibaba DashScope Starter |

架构上，浏览器访问 Vue 管理端，经路由进入各业务页面；页面调用后端 REST 接口，Controller 层完成参数校验与流程状态更新，Service/Mapper 访问 MySQL；入住评估报告、解除协议等附件可通过上传接口保存；AI 模块通过 ChatModel 调用通义千问等 DashScope 能力，提供普通响应与 SSE 流式响应两种方式。系统未引入 Redis 缓存中间件、Activiti/Flowable 工作流引擎等组件，入退住与请假流程由业务表字段（如 `step`、`flowStatus`）与协同接口自行编排。

## 1.6 核心业务流程简述

### （1）入住流程（五步）

入住业务由「入住申请」发起，在「入住办理」中跟踪，步骤组件定义为：

**申请入住 → 入住评估 → 入住审批 → 入住配置 → 签约办理**

申请阶段采集老人基本信息并生成入住单；评估阶段完成健康评估与能力评估并形成评估报告；审批节点可选择通过、拒绝或驳回；配置阶段确定护理等级、床位、费用与服务周期等；签约办理完成后流程结束，形成在住合同相关信息。流程中禁止无序跳步，已完成或已关闭单据不可继续推进。

### （2）退住流程（七步）

退住业务由「退住申请」发起，在「退住办理」中跟踪，步骤定义为：

**申请退住 → 申请审批 → 解除合同 → 调整账单 → 账单审批 → 退住审批 → 费用清算**

其中申请审批、账单审批、退住审批为审批节点（支持通过、拒绝、退回）；解除合同需填写解除日期并上传解除协议；调整账单可核定退款金额；费用清算为流程收尾，完成后退住单状态置为已完成。

### （3）协同工作

协同模块将入住、退住、请假等在途与历史单据统一汇总：「我的待办」侧重筛选申请中、待处理事项；「我的申请」用于查看本人发起单据及状态。用户可按单据类型、状态等条件检索，并跳转至对应详情页继续办理，从而形成跨模块的流程协同入口。

---

综上所述，中州养老管理系统以 Web 管理端承载养老机构日常运营业务，围绕入退住闭环与协同审批构建核心能力，并在 Spring Boot + Vue 技术体系上完成可演示、可扩展的课程实践成果。
